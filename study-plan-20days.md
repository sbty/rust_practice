# Rust 20日間 学習プラン

## ねらい

- 前半で Rust の構文と所有権をしっかり理解する
- 中盤で `cargo`、テスト、エラーハンドリング、モジュール設計を身につける
- 後半で小さなプロダクトを作りながら、GitHub を使ったモダンな開発手法に慣れる
- 最終的に「学ぶ」だけでなく「継続して作れる」状態に入る

## 学習の進め方

- 毎日 90 分から 150 分を目安に進める
- 各日の最後に `git commit` して学習ログを残す
- コードは「動くだけ」で終わらせず、必ず 1 本以上テストを書く
- わからない内容は `notes/` にメモし、翌日に 5 分で振り返る

## 推奨ディレクトリ構成

```text
rust_practice/
  notes/
  exercises/
    day01_hello_rust/
    day02_variables_and_types/
    ...
  products/
    cli_task_manager/
  README.md
  study-plan-20days.md
```

## 20日プラン

### Day 1: 環境構築と Rust の入口

- `rustup`、`cargo`、`rustc` の役割を把握する
- `cargo new exercises/day01_hello_rust`
- `fn main()`, `println!`, 変数宣言を試す
- `cargo run`, `cargo check`, `cargo build` の違いを確認する
- Git 初期化と最初のコミットを作る

成果物:
- 自己紹介を表示する簡単な CLI
- `notes/day01.md`

### Day 2: 変数、型、制御構文

- `let`, `mut`, シャドーイングを理解する
- 整数、浮動小数点、真偽値、文字、タプル、配列を触る
- `if`, `loop`, `while`, `for` を比較しながら使う
- 小さな計算問題を 3 問解く

成果物:
- `exercises/day02_variables_and_types`
- 偶数判定、合計計算、FizzBuzz

### Day 3: 関数と式指向

- 関数定義、引数、戻り値、式と文の違いを理解する
- ブロック式で値を返す感覚を掴む
- ユーティリティ関数を分けて書く練習をする

成果物:
- 温度変換、BMI 計算、最大値取得の関数群
- 関数ごとの単体テストを 2 本以上

### Day 4: 所有権と借用の基礎

- move、clone、参照、可変参照を理解する
- Rust がなぜ安全なのかを所有権から説明できるようにする
- コンパイルエラーを恐れず、意図的に壊して学ぶ

成果物:
- 所有権エラーの再現コードと修正版
- `notes/day04_ownership.md`

### Day 5: スライス、文字列、コレクション入門

- `String` と `&str` の違いを理解する
- スライス、`Vec<T>`、基本的な `String` 操作を学ぶ
- 文字列処理の小課題を解く

成果物:
- 文字数カウント、単語分割、最長単語取得
- テストで入力パターンを増やす

### Day 6: `struct` と `enum`

- 独自型でデータを整理する
- `match` を使った分岐を練習する
- `Option` と `Result` の前提になる考え方を掴む

成果物:
- TODO アイテムやユーザー情報を表す `struct`
- ステータス管理を表す `enum`

### Day 7: `match`、`Option`、`Result`

- null 安全の代わりに `Option` を使う考え方を学ぶ
- 失敗を `Result` で返す設計に触れる
- `unwrap` に頼りすぎない書き方を意識する

成果物:
- 簡単な入力バリデータ
- 正常系と異常系のテスト

### Day 8: モジュール分割と crate の理解

- `mod`, `pub`, `use` を使ってファイル分割する
- `main.rs` に書きすぎない設計を試す
- ライブラリ crate とバイナリ crate の違いを知る

成果物:
- 複数ファイル構成の小さな CLI
- `src/lib.rs` にロジック、`src/main.rs` に入出力

### Day 9: テスト駆動の入口

- `cargo test` を日常的に使う
- unit test と integration test の違いを学ぶ
- まずテストを書いてから実装するミニ課題を 1 つやる

成果物:
- `tests/` を持つ練習プロジェクト
- 失敗するテストから始めた実装履歴

### Day 10: エラーハンドリングとリファクタリング

- `?` 演算子、`thiserror` や `anyhow` の用途を把握する
- エラーメッセージを利用者目線で整える
- Day 8 か Day 9 のコードをリファクタリングする

成果物:
- エラー型を持つ小さなアプリ
- Before/After を `notes/day10_refactor.md` に記録

### Day 11: 標準ライブラリを使った CLI 強化

- ファイル入出力、`std::fs`、`std::path` を使う
- 引数処理の基礎を理解する
- JSON かテキストファイルの読み書きを試す

成果物:
- メモを保存できる CLI の原型
- ファイル読み書きのテスト

### Day 12: GitHub ベースの開発フロー入門

- GitHub にリポジトリを作る
- `main` 保護の考え方、Issue、Pull Request の流れを学ぶ
- 学習でもブランチを切って PR ベースで進める

やること:
- `feature/day12-cli-improve` のようなブランチで作業
- PR テンプレートを簡単に作る
- コミットメッセージを `feat:`, `fix:`, `test:` で揃える

### Day 13: 外部 crate を使う

- `clap`, `serde`, `serde_json` のような定番 crate に触れる
- 依存追加と `Cargo.toml` の読み方に慣れる
- 「自前で全部書かない」感覚を身につける

成果物:
- 引数を受け取って JSON 保存する CLI
- README に使い方を記述

### Day 14: プロダクト開始 - CLI タスク管理ツール

- `products/cli_task_manager` を新規作成
- 要件を絞って MVP を決める
- まずは以下の 4 機能だけ作る

対象機能:
- タスク追加
- タスク一覧表示
- タスク完了
- タスク削除

### Day 15: 設計とテストを整える

- ロジックと表示を分離する
- repository 的な層を簡単に分ける
- テストしづらい設計を改善する

成果物:
- `src/domain`, `src/app`, `src/infra` など軽い分割
- コアロジックの単体テスト充実

### Day 16: ユーザー視点の改善

- エラーメッセージ、ヘルプ表示、使いやすさを見直す
- サンプルデータや初回起動導線を考える
- README を「他人が使える」形にする

成果物:
- 実行例を含む README
- CLI の UX 改善

### Day 17: 品質向上のための自動化

- `cargo fmt`, `cargo clippy` を導入する
- GitHub Actions で test, fmt, clippy を回す
- CI が通る状態を維持する感覚を身につける

成果物:
- `.github/workflows/ci.yml`
- ローカルと CI の両方で品質チェック

### Day 18: 追加機能を 1 つ実装する

- 優先度、期限、タグ、検索のどれか 1 つを追加する
- 機能追加前に Issue を切る
- 小さく設計して PR でマージする

成果物:
- 機能追加の PR
- 仕様変更に合わせたテスト更新

### Day 19: 仕上げとリリース準備

- 不要コード削除、命名改善、例外ケースの確認を行う
- バージョン番号、CHANGELOG、リリースノートの雛形を作る
- 「人に見せるプロジェクト」に整える

成果物:
- `README.md`
- `CHANGELOG.md`
- 初回リリース候補

### Day 20: 振り返りと次の学習への接続

- 20 日間の学びを振り返る
- 苦手分野を 3 つ明確にする
- 次に進むテーマを選ぶ

候補:
- `tokio` で非同期処理
- `axum` で Web API
- `sqlx` や `diesel` で DB 連携
- `ratatui` で TUI アプリ
- `bevy` でゲーム開発

成果物:
- `notes/retrospective.md`
- 次の 30 日プランの下書き

## 週ごとの到達目標

### 1週目

- Rust の構文と所有権に慣れる
- コンパイラエラーを読みながら直せるようになる

### 2週目

- モジュール、テスト、エラーハンドリングを使って小さく設計できる
- GitHub を使って変更を管理できる

### 3週目

- 小さな CLI プロダクトを自力で進められる
- CI、README、PR ベースの進め方まで体験できる

## 毎日の共通チェックリスト

- 今日学んだ Rust の概念を 3 行で説明できるか
- `cargo fmt` と `cargo test` を実行したか
- 1 コミット以上残したか
- 明日のためのメモを 3 行残したか

## おすすめの進め方

- Day 1 から Day 10 は「広く浅く」ではなく「狭く深く」進める
- Day 11 以降は完成度よりも、設計変更と改善の反復を大事にする
- わからない概念が出たら、その日のうちに 100 点を目指さず 60 点理解で先に進む
- 週末か余裕のある日に、過去のコードを必ず 1 回リファクタリングする
